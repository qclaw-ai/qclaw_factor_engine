# 因子侧「真分域」因子值 — 设计与落地步骤

> **目标**：在 **因子工厂（qclaw_factor_engine）** 内，对指定股票池（领域）**单独计算并落盘**因子值（CSV 等），使 **截面 rank / 标准化 / 中性化** 等算子与 **该域成分** 一致；**多域可并存**。  
> **边界（已定）**：**策略工厂暂不改造**——策略回测/选股仍按 **单一默认因子值来源**（约定为 **`ALL` / 全市场** 路径）读取；真分域产物用于 **研究、入库实证、与默认路径对比**，不在本文档范围内实现「每因子选域拼组合」。  
> **前置**：已完成《多领域因子回测改造_落地步骤.md》中的库表与 `test_universe` 标识；本文在其之上 **增加「按域的因子值文件」**，二者需 **枚举与域编码对齐**。

---

## 0. 术语与关系

| 术语 | 含义 |
|------|------|
| **`test_universe` / 领域** | 与现有一致：`ALL`、`STOCK`、`HS300`、`ZZ500` 等，**因子值分域与回测分域共用同一套编码**。 |
| **真分域因子值** | 在该域 **当日可交易/成分并集** 上完成 DSL 要求的截面运算后得到的 **逐票因子序列**；**不同域 = 不同 CSV（或不同路径）**。 |
| **默认策略读取域** | 固定 **`ALL`**（或配置项 `default_factor_values_universe`），策略侧只认此域下的主路径，直至策略产品升级。 |
| **标识多域（已落地）** | `factor_backtest` 多行、`factor_universe_status`；**若回测读的是真分域 CSV，则 IC 与因子定义一致**；若仍读 ALL CSV 只改标签，则与 JQ 指数内语义仍可能不一致。 |
| **历史兼容** | 历史 `ALL_A` 统一映射为 `ALL`；新写入不再使用 `ALL_A`。 |

---

## 1. 设计原则

1. **路径可发现**：任意自动化任务可通过 `(factor_id, universe, 用途)` 定位文件，不依赖「目录里只有一个 csv」。  
2. **DB 可查询**：因子值路径 **不进 `factor_backtest`**（该行已有 `result_json_rel_path`）；建议 **独立表或结构化列** 存「批量因子值 / 日更」按域路径，避免 `factor_files` 单 PK `factor_id` 无法表达多域。  
3. **成分数据源**：域内运算依赖 **交易日 T 的成分股列表**（与 `data_ingest` / 指数成分源一致）；需定义 **缺失成分时**（停牌、未上市）是否剔除或前向填充。  
4. **与策略解耦**：策略仓库 **不修改**；仅在本文档 **§7** 写明「策略仍读默认域」的运维约定。  
5. **渐进交付**：先 **少量因子 × 少量域** 离线跑通，再扩日更与全量。

## 2. 文件与目录约定（建议）

以下路径均相对 **仓库根**（POSIX），与现有 `factor_files` 约定一致。

### 2.1 批量/大回测用因子值（按域）

统一方案（已定）：  
`factor_values/by_universe/{UNIVERSE}/{factor_id}_{UNIVERSE}_{start}_{end}.csv`  
例：`factor_values/by_universe/HS300/JQ_ALPHA_000_HS300_2025-01-01_2025-12-31.csv`  

说明：**ALL 也必须进入 `by_universe/ALL/`**，不再使用旧扁平路径。

### 2.2 日更因子值（按域）

若日更也要分域：  
`factor_values/daily/by_universe/{UNIVERSE}/{trade_date}/{factor_id}.csv`  
（或等价结构；**默认策略只同步 `ALL`** 子树到下游亦可。）

### 2.3 与现有 `backtest_results` 的关系

- **回测结果 JSON**：统一为  
  `backtest_results/by_universe/{UNIVERSE}/{factor_id}_{UNIVERSE}_backtest.json`（已实现）。  
- **真分域**后，JSON 内可注明 **`factor_values_rel_path`**（可选，便于审计），与 DB 路径表一致。

## 3. 数据库（因子库）建议改动

> **背景**：当前 `factor_files` 主键为 `factor_id`，**单行**难以承载「同因子多域 CSV + 日更多域」。

### 3.1 推荐：新表 `factor_value_files`（名称可微调）

| 列 | 说明 |
|----|------|
| `id` | serial PK |
| `factor_id` | FK → `factor_basic` |
| `universe` | varchar，与 `test_universe` 同枚举 |
| `artifact_type` | varchar：`batch_csv` / `daily_csv` / 预留 |
| `rel_path` | varchar(1024)，相对仓库根 |
| `date_start` / `date_end` | 可选，批量区间；日更可空或只填 `trade_date` |
| `created_at` | timestamp |

**唯一约束（示例）**：`(factor_id, universe, artifact_type, date_start, date_end)` 或日更用 `(factor_id, universe, trade_date)` 单独表——按产品粒度细化。

### 3.2 `factor_files` 兼容策略

- **保留**现有列：`doc_path`、`backtest_json_path`、`factor_values_path`（若有）、`factor_values_path_daily`（若有）。  
- **语义**：`factor_values_path` 建议明确为 **`default_factor_values_universe`（如 ALL）下的主批量路径** 或 **弃用为视图/同步字段**，避免与多域真值混淆。  
- **迁移**：新域路径 **只写 `factor_value_files`**，待稳定后再决定是否 **回写** `factor_files` 聚合字段。

### 3.3 与 `factor_backtest` 的衔接（可选）

- 可增加 **`factor_values_artifact_id`** 或 **`factor_values_rel_path`**（冗余）指向本次回测使用的 CSV，便于复现。  
- **非必须**：回测 runner 也可仅通过配置 `(factor_id, test_universe)` 拼路径。

## 4. 计算层（factor_engine / backtest_core）

### 4.1 引擎侧

1. **入口参数**：`universe`（或 `evaluation_universe`）传入 **因子计算管线**。  
2. **行情 panel**：仍可按 **全市场加载** 后 **按当日成分 mask**（省存储），或 **只加载域内股票**（省内存）；二选一写清。  
3. **截面算子**：`RANK`、`scale`、行业/市值中性等 **仅在 mask 后剩余股票上** 执行；与 JoinQuant 指数内定义对齐程度取决于 **成分表与可交易过滤** 是否与 JQ 一致。  
4. **输出**：写出 **该域** CSV，列格式与现有一致（`trade_date, stock_code, factor_value` 等），便于 `backtest_core` 复用。

### 4.2 `backtest_core`

1. **读盘**：`run_backtest_for_one`（或上层）根据 **`test_universe`** 解析 **真分域 CSV 路径**（配置 + 命名约定）。  
2. **IC/分层**：在 **同一域子集** 上算（或与因子列自然一致：CSV 仅含域内票则自动限制）。  
3. **与仅标识多域的区别**：此前可能用全市场 CSV + 未 mask；此处 **指标与因子定义一致**。

---

## 5. 批处理与 `backtest_io` / `selection_and_store`

1. **批跑**：对 `universe ∈ {ALL, HS300, ZZ500}` 循环调用因子引擎 → 多文件 → 多行 `factor_value_files`。  
2. **`backtest_io`**：插入 `factor_backtest` 时 **`test_universe` 与所用 CSV 域一致**；可选更新 `result_json_rel_path`。  
3. **`selection_and_store`**：逻辑 **已按域**；若 `factor_files.backtest_json_path` 仍指向 `primary_universe_for_file_pointer`，与现网一致。  
4. **`factor_basic.is_valid`**：保持 **任一侧 TRUE** 规则；真分域后各域 IC 可能分化，属预期。

---

## 6. 日更线（`daily_factor_values`）

1. **默认**：仅 **`ALL`** 日更 → 策略现状不变。  
2. **扩展**：增加参数 `--universe` 或配置列表；输出到 **§2.2** 目录；**仅写 `factor_value_files`**（`artifact_type=daily_csv`）。  
3. **调度**：多域日更任务量 = 因子数 × 域数，需 **队列/错峰**。

---

## 7. 策略侧约定（本文不开发）

- 策略配置、回测 runner **继续只解析默认域** 的因子值路径（由因子库 **约定文档 + DB 视图** 提供，例如始终 `ALL`）。  
- 清单中 **`evidence_universe`** 可为 HS300，表示 **实证依据**；**执行仍用 ALL 因子值** 时，产品文档需写明 **外推**，与《多领域因子回测改造》一致。  
- 日后策略要 **按因子选域读 CSV**，单独立项，不在本文范围。

## 8. 落地阶段（建议顺序）

| 阶段 | 内容 | 验收 |
|------|------|------|
| **P1** | 迁移：`factor_value_files`（或等价）；更新 `schema_mvp.sql` + `docs/db_conventions.md` | 空表可插入；约束正确 |
| **P2** | 成分服务：按 `trade_date` + `universe` 返回 `stock_code` 集合（复用/抽取 `data_ingest`） | 与手工抽样一致 |
| **P3** | `factor_engine`：universe 上下文 + mask 后截面 | 同一因子 ALL vs HS300 两行 CSV 不同（对含 RANK 因子明显） |
| **P4** | 离线批：3 因子 × 2 域 端到端 + `backtest_core` + `backtest_io` | DB 路径表 + `factor_backtest` 两行 IC 可区分 |
| **P5** | `selection` 联调；可选日更多域 | `factor_universe_status` 与真 IC 一致 |
| **P6** | 全量策略由产品决定；默认不扩策略代码 | 运维文档更新 |

## 9. 风险与回滚

- **成分偏差**：与 JQ/聚源不一致会导致「仍对不齐论文」——应用 **数据源版本与日期** 记录。  
- **存储与耗时**：域数倍增 → 监控磁盘与批处理时长。  
- **回滚**：停写 `factor_value_files`；恢复读取旧扁平路径需要单独代码回退；DB 新表可保留不删。
